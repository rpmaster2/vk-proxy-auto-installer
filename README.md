## ⚙️ Возможности автоустановщика
- 📦 Автоматически скачивает необходимые пакеты (`curl`, `wget`, `jq`, `ufw`, `qrencode`) для Debian/Ubuntu (также есть поддержка CentOS/RHEL, но скрипт там не обкатывался, делайте всё на свой страх и риск!)
- 🧠 Поддерживает установку разных ядер прямо с репозиториев авторов (список постепенно дополняется)
- 🛡️ Может с нуля установить WireGuard (через скрипт от angristan), AmneziaWG (через скрипт от wiresock) или автоматически найти порты уже существующих конфигураций WG/AWG
- 🎯 Поддерживает ручной ввод локального порта, что позволяет использовать прокси поверх уже установленных WireGuard, AmneziaWG, Hysteria2, xray/sing-box панелей
- 🚀 Автоматически создает systemd-службу для работы в фоне и открывает нужные внешние порты в UFW

## ✨ Возможности панели (`vk-panel`)
- 🟢 Запуск / 🔴 Остановка / 🔄 Перезапуск прокси
- 📥 Обновление установленного ядра с GitHub в один клик
- ➕ Добавление и удаление клиентов WireGuard/AmneziaWG
- 📱 Вывод QR-кодов существующих клиентов WireGuard/AmneziaWG прямо в терминал
- ⚙️ Автоматическое обновление самой панели
- ↔️ Смена ядра vk-turn-proxy в любой момент
- 🔌 Смена внешнего и локального портов
- 🛡️ Включение/отключение режима `-vless` (только для некоторых ядер)
- 📞 Включение/отключение режима Telemost DataChannel (`-telemost-dc`) (только для некоторых ядер)
- ✍️ Возможность использования своей команды запуска (режим Raw)
- 🗑️ Безопасное и полное удаление панели и ядра vk-turn-proxy

## 🛠 Установка

Запустите команду ниже на вашем сервере (рекомендуется чистая ОС Ubuntu/Debian):

```bash
bash <(curl -sL https://raw.githubusercontent.com/NedgNDG/vk-proxy-auto-installer/main/install.sh)
```
Для установки конкретной версии используйте команду ниже (вместо v1.0 прописывайте номер конкретной версии):

```bash
bash <(curl -sL https://raw.githubusercontent.com/NedgNDG/vk-proxy-auto-installer/v1.0/install.sh)
```

Далее скрипт сделает всё автоматически, вам надо лишь следовать подсказкамю

Для открытия панели просто пропишите `vk-panel`

## 📱 Как подключиться и настроить клиент на телефоне

Далее будут описаны способы подключения с использованием утилиты [TURN Proxy от MYSOREZ](https://github.com/MYSOREZ/vk-turn-proxy-android) в режме своей (Raw) команды с использованием стороннего клиента (подойдёт любой sing-box/xray клиент) только под андроид. С каждым днём приложений и способов обхода появляется всё больше и больше, поэтому будет описан самый базовый и универсальный, однако ниже будут рекомендации приложений, подробней в репозиториях авторов. Перед этим настройте серверную часть через меню и используйте соответствующие аргументы в клиенте.

**В приложении TURN Proxy:**

0. Перед настройкой скачайте нужное Android ядро в одном из репозиториев авторов, в приложении нажмите **Загрузить новое ядро (файл)** и выберете скачанный файл
1. В настройках переключите "Режим своей (Raw) команды", чтобы он стал активен
2. Вписываем:
```bash
./client -listen 127.0.0.1:9000
```
  a) Далее если используете VK Call Links
```bash
-n 4 -peer ВАШ_IP_АДРЕС_СЕРВЕРА:IP_VK-TURN-PROXY (по умолчанию 56000) -vk-link ССЫЛКА_VK_CALL
```
  b) Далее если используете Yandex Telemost DC
```bash
-n 1 -yandex-link  ССЫЛКА_YANDEX_TELEMOST (используйте ту же, что и на сервере) -telemost-dc
```
Флаги `-vless` (должен быть включён на сервере, рекомендуются использовать под него 443/8443 порты) и `-udp` используйте от ваших предпочтений или если другой режим не работает

3. Сохраняем изменения и запускаем.

Примечание: данные команды также должны подойти и для клиента на пк, подробнее можете узнать [тут](https://github.com/cacggghp/vk-turn-proxy?tab=readme-ov-file#windows).

**В приложении с sing-box/xray ядром:**

1. Отсканируйте QR-код вашего конфига (будь то созданный через данную панель или стороннюю через web-панель). НЕ ВКЛЮЧАЕМ!
2. В блоке Адрес смените айпи на 127.0.0.1, в блоке Порт смените порт на 9000
3. Поставьте MTU 1280 (если выбран WireGuard)
4. Сохраняйте
5. Добавляем TURN Proxy в исключения

Сохраните и подключитесь!

Альтернативные клиенты:
- Модифицированный клиент WireGuard [WG Turn от kiper292](https://github.com/kiper292/wireguard-turn-android) (особенно рекомендуется использовать с ядром на сервере от самого [kiper292](https://github.com/kiper292/vk-turn-proxy))
- [WINGSV от WINGS-N](https://github.com/WINGS-N/WINGSV) (особенно если хотите использовать с AmneziaWG на сервере)
- [FreeTurn от samosvalishe](https://github.com/samosvalishe/turn-proxy-android) (имеет возможность установки ядра на сервер прямо из клиента)
- [VkPN от oxsidee](https://github.com/oxsidee/vkpn) (имеются версии как для мобильных, так и для стационарных OS)
- [WDTT от amurcanov](https://github.com/amurcanov/proxy-turn-vk-android) 

P.S. Способы подключения могут быть взаимозаменямы, несмотря на то, чью серверную часть вы используйте, но при обновлении всё может измениться.
P.S.S. Данный метод обхода по сути является гонкой вооружений, сегодня может работать одно, завтра другое, а послезавтра может сломаться всё, поэтому настоятельно рекомендую следить за репозиториями авторов, своевременно обновляться и пробовать разные методы, если какой-то не работает.

## ✔️ Рекомендации
- Не используйте данный метод обхода для загрузки торрентов и гигабайтов данных
- Используйте точечную маршрутизацию (настраивается в самом конфиге)
- Не используйте один и тот же конфиг, и одну и ту же ссылку на звонок на разных устройствах
- Старайтесь использовать по необходимости в период работы белых списков
