## ⚙️ Возможности автоустановщика
- 📦 Автоматически скачивает необходимые пакеты (`curl`, `wget`, `jq`, `ufw`, `qrencode`) для Debian/Ubuntu (также есть поддержка CentOS/RHEL, но скрипт там не обкатывался, делайте всё на свой страх и риск!)
- 🧠 Поддерживает установку разных ядер прямо с репозиториев авторов (список постепенно дополняется)
- 🛡️ Может установить WireGuard "с нуля" (через скрипт от angristan), AmneziaWG (через скрипт от wiresock) или автоматически найти порты уже существующих конфигураций WG/AWG
- 🎯 Поддерживает ручной ввод локального порта, что позволяет использовать прокси поверх уже установленных WireGuard, AmneziaWG, Hysteria2, Xray-панелей
- 🚀 Автоматически создает systemd-службу для работы в фоне и открывает нужные внешние порты в UFW

## ✨ Возможности панели (`vk-panel`)
- 🟢 Запуск / 🔴 Остановка / 🔄 Перезапуск прокси
- 📥 Умное автообновление прокси с GitHub в один клик
- ➕ Добавление и удаление клиентов WireGuard/AmneziaWG
- 📱 Вывод QR-кодов существующих клиентов WireGuard/AmneziaWG прямо в терминал
- ⚙️ Автоматическое обновление самой панели
- ↔️ Смена ядра vk-turn-proxy в любой момент
- 🔌 Смена внешнего и локального портов
- 🗑️ Безопасное и полное удаление vk-turn-proxy

## 🛠 Установка

Запустите команду ниже на вашем сервере (рекомендуется чистая ОС Ubuntu/Debian):

```bash
bash <(curl -sL https://raw.githubusercontent.com/NedgNDG/vk-proxy-auto-installer/main/install.sh)
```
Для установки конкретной версии используйте команду ниже (вместо v1.0 прописывайте номер конкретной версии):

```bash
bash <(curl -sL https://raw.githubusercontent.com/NedgNDG/vk-proxy-auto-installer/v1.0/install.sh)
```

Далее скрипт сделает всё автоматически, вам надо лишь следовать подсказкамю

Для открытия панели просто пропишите `vk-panel`

## 📱 Как подключиться и настроить клиент на телефоне

На выбор имеются несколько вариантов подключения:
1. Через модифицированный клиент WireGuard [WG Turn от kiper292](https://github.com/kiper292/wireguard-turn-android) (особенно рекомендуется использовать с ядром на сервере от самого [kiper292](https://github.com/kiper292/vk-turn-proxy))
2. Через приложение NekoBox ([рекомендую мод от Starifly](https://github.com/starifly/NekoBoxForAndroid)) (или другой клиент с xray или sing-box ядром) и утилитой [vk-turn-proxy-android от MYSOREZ](https://github.com/MYSOREZ/vk-turn-proxy-android) (особенно если собираетесь использовать с vless)
3. Через [клиент от WINGS-N](https://github.com/WINGS-N/WINGSV), особенно если хотите использовать с AmneziaWG на сервере

### Вариант первый (Только WireGuard):

На сервере запустите vk-panel, выберите пункт 7; или откройте созданный конфиг WireGuard на веб-панели

**В приложении WG Turn:**

1. Отсканируйте QR-код вашего конфига. НЕ ВКЛЮЧАЕМ!
2. В блоке Пир измените Endpoint (Конечная точка) на 127.0.0.1:9000
3. Поставьте MTU 1280
4. Включите галочку Enable TURN Proxy, Peer type оставьте Proxy v2
5. Достаньте ссылку звонка ВК и вставьте в строку VK Call links. ВАЖНО!!! Создайте для этих целей твинк аккаунт. Ни в коем случае не нажимаем "Завершить звонок для всех", достаточно просто выйти из него, он никуда не пропадёт
6. В строку Peer впишите: ВАШ_IP_АДРЕС_СЕРВЕРА:IP_VK-TURN-PROXY (по умолчанию 56000)
7. Локальный порт пропишите: 9000
8. В строку Streams (Потоки) прописываем 4
9. UDP используйте по желанию
10. Сохраните и подключитесь, вылезет капча. Решите её

Готово!

Также в последних версиях в приложении добавлен WB Mode. Если на сервере установлена реализация от kiper292, можете выбрать его, в таком случае даже не понадобиться ссылка VK Call, а значит шаг 5 можно смело пропустить

### Вариант второй:

**В приложении VK TURN Proxy:**

0. Перед настройкой скачайте нужное Android ядро в одном из репозиториев авторов, в приложении нажмите **Загрузить новое ядро (файл)** и выберете скачанный файл
1. В строку Peer впишите: ВАШ_IP_АДРЕС_СЕРВЕРА:IP_VK-TURN-PROXY (по умолчанию 56000)
2. Достаньте ссылку звонка ВК и вставьте в строку VK Call links. ВАЖНО!!! Создайте для этих целей твинк аккаунт. Ни в коем случае не нажимаем "Завершить звонок для всех", достаточно просто выйти из него, он никуда не пропадёт
3. В строку Streams (Потоки) прописываем 4
4. UDP используйте по желанию
5. Запускайте
6. Обратите внимание на терминал логов, там появится ссылка, перейдите по ней и решите капчу

ВАЖНО!!! Если при установке на сервере вы выбрали ядро от [Moroka8](https://github.com/Moroka8/vk-turn-proxy) на сервере автоматически прописывается флаг -vless. При его использовании скачайте клиентскую версию ядра с репозитория автора, в приложении VK TURN Proxy загрузите скачанное ядро, активируйте "Режим своей (Raw) команды и добавьте флаг `-vless` на конец

Пример команды
```bash
./client -n 4 -peer 123.45.67.89:56000 -vk-link ССЫЛКА_VK_CALL -listen 127.0.0.1:9000 -vless
```

На сервере запустите vk-panel, выберите пункт 7; или откройте созданный конфиг WireGuard/VLESS на веб-панели

**В приложении NekoBox:**

1. Отсканируйте QR-код вашего конфига. НЕ ВКЛЮЧАЕМ!
2. В блоке Адрес смените айпи на 127.0.0.1, в блоке Порт смените порт на 9000
3. Поставьте MTU 1280 (если выбран WireGuard)
4. Сохраняйте
5. Добавляем VK TURN Proxy в исключения

Сохраните и подключитесь!

### Вариант третий (Способ через backend VK TURN + AmneziaWG):

0. С сервера выгрузите готовый AWG конфиг на телефон с установленным WINGSV
1. В настройках в пункте Активный backend выберите VK TURN + AmneziaWG
2. Скопируйте текст вашего скаченного конфига, выберете пункт Настройки VK TURN + AmneziaWG и импортируйте из буфера обмена, всё должно примениться автоматически
3. В блоке Endpoint впишите: ВАШ_IP_АДРЕС_СЕРВЕРА:IP_VK-TURN-PROXY (по умолчанию 56000)
4. Достаньте ссылку звонка ВК и вставьте в строку VK Link. ВАЖНО!!! Создайте для этих целей твинк аккаунт. Ни в коем случае не нажимаем "Завершить звонок для всех", достаточно просто выйти из него, он никуда не пропадёт
5. В строку Потоки прописываем 4
6. UDP используйте по желанию
7. Локальный endpoint оставьте 127.0.0.1:9000

Сохраните и подключитесь! При подключении вылезет окно с капчей, решите её. Проверьте уведомления, скорее всего понадобиться решить ещё несколько капч. После этого всё должно работать.

P.S. Способы подключения могут быть взаимозаменямы, несмотря на то, чью серверную часть вы используйте, но при обновлении всё может измениться

## ✔️ Рекомендации
- Не используйте данный метод для загрузки торрентов и гигабайтов данных.
- Используйте точечную маршрутизацию (настраивается в самом конфиге).
- Не используйте один и тот же конфиг, и одну и ту же ссылку на звонок на разных устройствах.
- Старайтесь использовать по необходимости в период работы белых списков.
